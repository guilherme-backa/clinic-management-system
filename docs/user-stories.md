# User Stories

**Versão**: 1.0  
**Data**: 2026-04-28  
**Status**: Draft

---

## Objetivo

Documentar as principais histórias de usuário por perfil para orientar backlog, refinamento funcional e priorização do desenvolvimento.

---

## Super Admin

### US-SA-001
Como **Super Admin**, quero visualizar todos os workspaces da plataforma para conseguir administrar a operação global.

### US-SA-002
Como **Super Admin**, quero acessar dados administrativos de qualquer workspace para oferecer suporte e resolver incidentes.

### US-SA-003
Como **Super Admin**, quero consultar logs globais para auditar atividades relevantes da plataforma.

### US-SA-004
Como **Super Admin**, quero suspender um workspace quando necessário para proteger a plataforma.

---

## Workspace Owner

### US-WO-001
Como **Workspace Owner**, quero criar meu workspace para iniciar a operação da minha organização.

### US-WO-002
Como **Workspace Owner**, quero cadastrar a primeira clínica obrigatoriamente para começar a usar o sistema.

### US-WO-003
Como **Workspace Owner**, quero cadastrar novas clínicas para administrar múltiplos estabelecimentos.

### US-WO-004
Como **Workspace Owner**, quero convidar usuários para o workspace para distribuir responsabilidades.

### US-WO-005
Como **Workspace Owner**, quero visualizar os usuários vinculados ao workspace para gerenciar acessos.

### US-WO-006
Como **Workspace Owner**, quero definir ou ajustar papéis para garantir segurança e controle.

---

## Establishment Owner

### US-EO-001
Como **Establishment Owner**, quero editar dados da clínica para manter as informações corretas.

### US-EO-002
Como **Establishment Owner**, quero vincular usuários ao meu estabelecimento para montar minha equipe.

### US-EO-003
Como **Establishment Owner**, quero visualizar os profissionais da clínica para organizar a operação.

### US-EO-004
Como **Establishment Owner**, quero consultar a atividade do estabelecimento para acompanhar o uso do sistema.

---

## Administrativo

### US-AD-001
Como **Administrativo**, quero acessar dados cadastrais de usuários e clínicas para manter a operação organizada.

### US-AD-002
Como **Administrativo**, quero consultar convites pendentes para acompanhar o processo de entrada de novos usuários.

### US-AD-003
Como **Administrativo**, quero inativar usuários quando necessário para manter segurança operacional.

### US-AD-004
Como **Administrativo**, quero consultar informações consolidadas do workspace para apoiar a gestão.

---

## Recepcionista

### US-RC-001
Como **Recepcionista**, quero cadastrar um tutor para iniciar o atendimento.

### US-RC-002
Como **Recepcionista**, quero cadastrar um animal para permitir seu acompanhamento clínico.

### US-RC-003
Como **Recepcionista**, quero agendar uma consulta para organizar a agenda da clínica.

### US-RC-004
Como **Recepcionista**, quero remarcar ou cancelar agendamentos para refletir mudanças no atendimento.

### US-RC-005
Como **Recepcionista**, quero localizar rapidamente um tutor ou animal para agilizar o atendimento.

---

## Veterinário

### US-VT-001
Como **Veterinário**, quero visualizar minha agenda para organizar meus atendimentos.

### US-VT-002
Como **Veterinário**, quero abrir o histórico do animal para entender seu contexto clínico.

### US-VT-003
Como **Veterinário**, quero registrar um prontuário para documentar o atendimento.

### US-VT-004
Como **Veterinário**, quero registrar vacinação para manter o histórico de prevenção.

### US-VT-005
Como **Veterinário**, quero emitir uma prescrição para formalizar o tratamento indicado.

### US-VT-006
Como **Veterinário**, quero registrar exames e procedimentos para manter a rastreabilidade clínica.

---

## Funcionário / Auxiliar

### US-FN-001
Como **Funcionário**, quero acessar apenas as funcionalidades permitidas para meu papel para executar minhas tarefas com segurança.

### US-FN-002
Como **Funcionário**, quero visualizar informações operacionais necessárias ao meu trabalho para apoiar a clínica.

### US-FN-003
Como **Funcionário**, quero colaborar com a rotina da clínica sem acessar dados indevidos.

---

## Histórias Transversais

### US-TR-001
Como **usuário**, quero fazer login com segurança para acessar apenas o que tenho permissão.

### US-TR-002
Como **usuário**, quero recuperar minha senha para voltar a acessar o sistema quando necessário.

### US-TR-003
Como **usuário**, quero gerenciar meu perfil para manter meus dados atualizados.

### US-TR-004
Como **gestor**, quero que ações críticas sejam auditadas para garantir rastreabilidade.

---

## Critérios Gerais

Cada história deve, idealmente, ser refinada depois com:
- critérios de aceitação;
- regras de negócio;
- fluxos alternativos;
- tratamento de erro;
- impacto em permissões;
- impacto em auditoria.
